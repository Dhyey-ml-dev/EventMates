═══════════════════════════════════════════════════════════════════════
                      ADMIN API ENDPOINTS
═══════════════════════════════════════════════════════════════════════

All endpoints require: Authentication (JWT) + role='admin'

GET    /api/admin/dashboard/stats          → Get statistics
GET    /api/admin/users                    → List all users
PATCH  /api/admin/users/:userId/verify    → Verify user
PATCH  /api/admin/users/:userId/suspend   → Suspend user
GET    /api/admin/events                   → List all events
PATCH  /api/admin/events/:eventId/approve → Approve event
GET    /api/admin/reports                  → Get reports
PATCH  /api/admin/reports/:reportId/status → Update report
GET    /api/admin/analytics/payments       → Payment analytics
GET    /api/admin/logs                     → System logs

═══════════════════════════════════════════════════════════════════════
                          HOW TO USE
═══════════════════════════════════════════════════════════════════════

STEP 1: Ensure both servers are running
   Terminal 1: npm run dev (in /server)
   Terminal 2: npm run dev (in /client)

STEP 2: Access admin dashboard
   URL: http://localhost:5173/admin/dashboard

STEP 3: Login with admin credentials
   • Create an admin account, OR
   • Use role='admin' in any user object

STEP 4: Navigate the dashboard
   • Click tabs to explore different sections
   • Approve/reject users and events
   • Review reports and logs
   • Monitor system health
